Konfiguracja alertów Resend

Instrukcja uruchomienia e-maili ostrzegających o niskim saldzie SMSAPI i OpenRouter. Do wykonania 23 lipca 2026.

Status: wymaga konfiguracji Czas: około 15–30 minut Bez umieszczania klucza w kodzie
Nadawcaformularz@widocznosc.ai
Odbiorcam.wisniewski@grupa-icea.pl
Brakujący elementRESEND_API_KEY
Ważne: klucza zaczynającego się od re_ nie wklejamy do pliku, commita, wiadomości ani przeglądarkowego kodu aplikacji. Trafia wyłącznie do GitHub Actions Secrets.

Kroki do wykonania

1

Zaloguj się do Resend

Zaloguj się na istniejące konto firmowe albo utwórz konto przeznaczone do wysyłek systemowych.

Otwórz Resend
2

Sprawdź domenę widocznosc.ai

  1. Otwórz sekcję Domains.
  2. Jeżeli widocznosc.ai ma status Verified, przejdź do kroku 3.
  3. Jeżeli domeny nie ma, kliknij Add Domain i dodaj widocznosc.ai.
  4. Skopiuj wygenerowane przez Resend rekordy DKIM, SPF i MX do panelu DNS domeny.
  5. Wróć do Resend i kliknij Verify DNS Records.

Nie używaj przykładowych rekordów z internetu — wartości DNS są generowane indywidualnie przez Resend.

Otwórz domeny Resend
3

Utwórz klucz tylko do wysyłki

W sekcji API Keys kliknij Create API Key i ustaw:

  • Name: Dashboard zaplecza
  • Permission: Sending access
  • Domain: widocznosc.ai

Skopiuj klucz re_... od razu — będzie widoczny tylko raz.

Otwórz API Keys
4

Dodaj sekret w GitHubie

  1. Otwórz ustawienia sekretów repozytorium.
  2. Kliknij New repository secret.
  3. W polu Name wpisz dokładnie RESEND_API_KEY.
  4. W polu Secret wklej klucz re_....
  5. Kliknij Add secret.
Otwórz sekrety repozytorium
5

Sprawdź odbiorców i progi

Konfiguracja znajduje się w dashboard/domains.yaml. Obecnie wiadomość trafi do jednego odbiorcy, a alert uruchomi się poniżej 20 SMS-ów lub 20 USD salda OpenRouter.

alerts:
  emails:
    - m.wisniewski@grupa-icea.pl
  webhook_envs:
    - ALERT_WEBHOOK_URL
  sms_min: 20
  openrouter_min_usd: 20

Kolejne adresy e-mail dodajemy jako następne pozycje pod emails.

6

Wykonaj kontrolowany test

Przy obecnych saldach zwykłe uruchomienie collectora nie wyśle wiadomości, ponieważ progi nie są przekroczone. Do jednorazowego testu:

  1. tymczasowo ustaw openrouter_min_usd: 9999,
  2. zapisz zmianę i poczekaj na zakończenie workflowu Dashboard Collector,
  3. sprawdź skrzynkę odbiorczą oraz folder spam,
  4. natychmiast przywróć właściwy próg, np. 20.
Otwórz workflow
Oczekiwany rezultat: wiadomość o tytule „⚠ Dashboard zaplecza: niski stan kredytów” zostanie wysłana z adresu formularz@widocznosc.ai. Kolejne alerty będą sprawdzane podczas przebiegów collectora.

Checklista końcowa

  • Domena widocznosc.ai ma w Resend status Verified.
  • Klucz ma uprawnienie Sending access, nie Full access.
  • W GitHub Actions istnieje sekret RESEND_API_KEY.
  • Testowa wiadomość dotarła do odbiorcy.
  • Po teście przywrócono normalny próg alertu.
  • Klucz nie został zapisany w repozytorium ani udostępniony osobom trzecim.
